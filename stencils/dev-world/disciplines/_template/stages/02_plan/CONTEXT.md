# <Discipline> / 02_plan

**Job:** Map the consequences of the change. Define the approach. No code changes.

## Inputs
- Target from 01_identify
- Blast radius (callers, dependents)

## Process
1. Define what the change needs to do (one sentence)
2. List what it must NOT break
3. Define the smallest change that achieves the goal
4. Note any edge cases or gotchas

## Outputs
- Plan written out: what will change, what will not, why
- Risk list: anything this touches that could break

## Gate
None. Read-only.
